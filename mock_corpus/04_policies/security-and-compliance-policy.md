# Security and Compliance Policy

## Available security controls

| Control | Starter | Business | Enterprise |
|---|---|---|---|
| SAML SSO | No | Yes | Yes |
| Audit logs | No | Yes | Yes |
| Role-based access control | Basic | Full | Full |
| Encryption at rest and in transit | Yes | Yes | Yes |
| IP allowlisting | No | No | Yes |
| Advanced security controls | No | No | Yes |
| Dedicated infrastructure | No | No | Add-on |

## Certifications and compliance posture

NorthstarCRM maintains standard B2B SaaS security practices. Security documentation including our security overview and data handling summary is available for active evaluations. Specific certification details (e.g., SOC 2 reports, penetration test results) are shared under NDA during active deal cycles.

The chatbot must not fabricate certifications, audit outcomes, or compliance commitments not documented here or in the data handling policy.

## Security questionnaires (TPRM)

Third-party risk management (TPRM) questionnaires and vendor security assessments are supported during active evaluations. Completing a TPRM questionnaire is a multi-step process:

1. The prospect provides the questionnaire to the Account Executive.
2. The AE submits it to the security team for review and completion.
3. Typical turnaround is 5–10 business days depending on questionnaire complexity.
4. TPRM questionnaires are not completed before the demo or during initial qualification conversations.

The chatbot cannot complete or pre-fill security questionnaires. Requests to complete a TPRM before the demo should be escalated to the Account Executive, who will set expectations on timing.

## Data handling

- Customer data is used only for service delivery.
- Data retention is 2 years by default; extended retention is available on Enterprise.
- Data deletion requests are handled by the support team per the data handling policy.
- No promise of data residency (e.g., specific geographic data storage) unless documented in the signed contract.

## Security addenda and custom commitments

Security addenda or custom commitments beyond what is documented here require review by the security and legal teams. The chatbot cannot pre-approve security addenda. See the legal review process for how custom terms are handled.
